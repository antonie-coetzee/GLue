Execute File




Opens and executes a Lua script from the provided path.

Parameter(s)
-----------------

- MLua in: Reference to the MLua instance

- Path: Path to the Lua file to be executed.

- Argument Data: LabVIEW variant to be used as argument data for the Lua script.

- ms timeout (-1): The time to wait for the MLua reference to become available. A value of '-1' specifies an infinite timeout.

- error in: Describes error conditions that occur before this node runs. This input provides standard error in functionality.

- MLua out: Reference to the MLua instance.

- Return Data: Return data from the Lua script in the form of a LabVIEW variant.

- error out: Contains error information. This output provides standard error out functionality.

 

Overview:

 

Runs the script that is specified by the Path parameter. The script it provided with Argument Data that is converted from the LabVIEW variant into the Lua equivalent. The Lua script can access this argument data using the 'vararg' parameter (...). During the scripts execution it may call the MLua API. When the Lua script execution ends, it may optionally return data. This data must conform to certain criteria for it to be correctly parsed back into LabVIEW data, explained in the MLua reference section.

 

Usage:

 

Below is shown how the Execute File VI is used. First a New Instance is created, followed by the Execute File VI which is provided with a path to the script to be executed. A LabVIEW cluster is provided as argument data and return data is converted to a LabVIEW string. Lastly the MLua instance is freed with the Close Instance VI.
